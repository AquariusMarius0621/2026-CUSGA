## ch10 开发规范
文档规范：
- 主记忆文档控制在 `1000` 行以内，超量拆到子文档
- 主文档只记录项目概况、文件索引、核心规则、游戏流程
- 架构、已知问题、历史、路线图进入子文档
- 每次实质性开发后，必须更新版本号、日期、索引、变更点

代码规范（本项目专用）：
- `【塔防开发】` 任务中的新脚本与改动脚本，保持高注释密度、中文讲解、教学语气
- 改逻辑必改注释，不允许代码行为和注释含义脱节
- 优先写“小而清楚”的组件，而不是把所有功能继续塞进总控
- 新增场景对象要么保持命名清晰，要么同步更新记忆文档

验证规范：
- 代码改动后至少做一次 `dotnet build Assembly-CSharp.csproj -nologo`
- 场景改动后至少执行：`scene_save`
- 涉及运行时逻辑时，至少检查一次 Play 模式启动 + `console_get_logs Error/Exception`
