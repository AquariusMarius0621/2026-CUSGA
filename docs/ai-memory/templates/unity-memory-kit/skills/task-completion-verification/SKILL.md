name: task-completion-verification
description: Verify multi-part `<PROJECT_TAG>` requests against the user checklist before wrapping up. Use when the request contains multiple independent deliverables such as code, scene, UI, docs, or validation.
---

1. Convert the request into a flat checklist.
2. Verify each item with concrete evidence from files, scene state, or validation steps.
3. Mark anything only partially verified.
4. Do not hide gaps behind a generic completion claim.
5. Group the final response by outcome rather than by raw file dump.
