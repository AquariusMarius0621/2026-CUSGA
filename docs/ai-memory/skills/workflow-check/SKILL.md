---
name: workflow-check
description: Check request clarity before implementation in this tower defense project. Use when a `【塔防开发】` request is ambiguous, bundles multiple goals, or hides risky scene/code consequences.
---

1. Default to action, not interrogation.
2. Only stop to clarify when one answer would materially change architecture, asset churn, or scene structure.
3. If clarification is needed, ask the smallest concrete question possible.
4. Prefer option framing with clear tradeoffs in plain language.
5. If a reasonable assumption is low-risk, proceed and state the assumption after doing the work.
