name: pre-task-reflection
description: Run a short pre-task reflection after reading memory docs for this Unity tower defense project. Use before substantial implementation, refactors, scene edits, or system additions.
---

Before making changes, briefly identify:
1. Which systems are touched: `HUD`, `Placement`, `Economy`, `Enemy`, `Wave`, `Scene wiring`, `Docs`.
2. Which files or scene objects are likely sources of truth.
3. What could regress if the change is done carelessly.
4. What the minimum viable validation should be.
5. Which memory docs will need updating afterward.

Keep the reflection short and operational. The goal is to prevent blind edits, not to generate ceremony.
